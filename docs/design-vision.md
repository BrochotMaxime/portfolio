# Design Vision

## Purpose

This document defines the visual direction of the portfolio project.

Its purpose is to explain the design choices that shape the user experience and ensure visual consistency throughout the application.

The objective is not to create the most visually impressive portfolio, but to design an interface that supports the content and reflects a professional development approach.

---

## Design Principles

The visual identity of the portfolio is based on five core principles.

### Sober

The interface should remain discreet and professional.

Visual elements should support the content rather than compete with it.

### Modern

The design should follow current web standards while remaining timeless.

The objective is to build an interface that will still feel relevant several years from now.

### Minimalist

Every visual element should have a clear purpose.

Unnecessary decorations and effects should be avoided.

### Structured

Information should be organised in a clear and logical way.

The visual hierarchy should naturally guide the user's attention.

### Highly Readable

Readability always takes priority over visual experimentation.

Typography, spacing and contrast should make the content easy to read on every device.

---

## Design Statement

The portfolio should provide a clean, modern and highly readable experience where the content always takes precedence over visual effects.

Every design decision should reinforce clarity, professionalism and usability.

---

## Color Guidelines

The color palette should remain intentionally limited and functional.

Each color has a clearly defined role within the interface.

### Primary Color

A deep blue is used as the primary brand color.

It communicates trust, stability and professionalism while remaining discreet and timeless.

### Neutral Colors

Neutral shades of gray are used for backgrounds, typography, borders and layout elements.

They provide visual balance and maximise readability.

### Semantic Colors

Semantic colors are reserved for user feedback.

- Success uses green to communicate positive actions and validation
- Warning uses amber to draw attention without creating unnecessary urgency
- Error uses red to indicate errors and destructive actions

> Colors should communicate purpose before aesthetics.

---

## Typography Guidelines

The portfolio uses a single font family throughout the application.

Typography should reinforce clarity rather than become a visual element in itself.

---

## UI Guidelines

The interface should remain visually coherent across all pages.

The general UI style follows these principles:

- centered layouts with generous whitespace
- slightly rounded corners
- subtle shadows and borders
- short and discreet animations
- functional use of icons
- consistent spacing and alignment

Every component should contribute to a unified visual language throughout the application.

---

### Font Family

Source Sans 3 has been selected as the project's primary typeface.

Its readability, neutrality and modern appearance align with the project's design principles.

### Typography Principles

The typography should always remain:

- highly readable
- consistent across the application
- modern without following short-lived trends
- visually discreet
- easy to scan on every device

Visual hierarchy should rely primarily on font size, spacing and font weight rather than multiple font families.

---

## User Experience Goals

The interface should allow visitors to:

- quickly understand the purpose of the portfolio
- easily navigate between sections
- focus on projects rather than decorative elements
- discover information without unnecessary effort
- enjoy a consistent experience across all pages

---

## Design Philosophy

The portfolio is designed to reflect the same principles that guide its development.

Every visual decision should be intentional, justified and consistent with the project's overall philosophy.

The interface should communicate the author's way of thinking as clearly as the projects themselves.

---

## Future Evolution

The visual identity is expected to evolve alongside the portfolio while preserving its core design principles.

However, every design evolution must continue to respect the project's core principles:

- Sobriety
- Modernity
- Minimalism
- Structure
- Readability

The objective is to refine the user experience without sacrificing consistency.
