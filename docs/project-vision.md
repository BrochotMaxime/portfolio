# Project Vision

This portfolio is designed as a professional platform that will evolve over time.

Its goal is not only to showcase completed projects, but also to demonstrate the way its author thinks, structures work and makes technical decisions.

## Main objective

Build a portfolio that demonstrates technical skills, architectural thinking and a professional development workflow, while remaining fully understood and maintained by its author.

## Project motto

> Make the portfolio show not only what has been developed, but also how its author thinks as a developer.

## Guiding principles

### 1. Every decision must be justified

No technology, library, design choice or feature should be added only because it is trendy.

Each decision must answer a clear question:

> Why is this choice relevant?

### 2. The code should reflect a way of working

The repository should show a professional development approach through:

* clear architecture
* reusable components
* consistent naming
* readable documentation
* maintainable code

### 3. The portfolio should evolve over time

The portfolio is not designed as a one-time project.

It should be easy to update with:

* new projects
* new skills
* professional experience
* improved design and features

### 4. Quality over quantity

A smaller number of well-designed features is better than many unnecessary additions.

Each feature should bring real value to the user or to a recruiter.

## Decision rule

If a feature does not bring value to a recruiter, it should not be developed.

## Learning objective

This project is also a learning tool.

Each technical decision should help its author progress while staying adapted to his current level.
